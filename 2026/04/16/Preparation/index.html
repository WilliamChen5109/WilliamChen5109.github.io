高性能计算

1. SIMD 和 SIMT 的区别

simd 指“单指令多数据”，一般用于cpu，面向处理数据组，simt 则指“单指令多线程”，一般指面向gpu的线程组。

2. CUDA 线程组织

GRID(网格)：一个核函数启动的所有线程的总和。

BLOCK(线程块)：一个 grid 可以分成多个 block ，每个 block 有唯一标识可通过 blockIdx.x，blockIdx.y，blockIdx.z 访问，同一个 block 内的线程可以访问同一块共享内存（shared memory），可使用 syncthreads() 进行同步，block是资源分配的最小单位，gpu 以 block 为单位将线程分配到 sm ，一个 sm 可以放多个 block，一个 block 会固定到分配的 sm 到程序结束。

THREAD(线程)：一个 block 里有多个 thread，thread 是最小的逻辑运行单位，可通过 threadIdx.x，threadIdx.y，threadIdx.z 进行访问。

3.warp的作用

GPU 内的线程被组织成每32个线程一组的 warp(线程束) ，因为在 SIMT 模式下，希望更多的线程能高效执行，如果每个线程都挨着启动统一指令，效率必然不高，如果所有线程同时启动同一指令，又丧失了灵活性，于是有了warp (线程束)。多个 warp 可以在 SM 上并行执行，同一个 warp 的线程执行同一条命令，同一个 warp 内的线程并行处理不同数据，从而在保证并行处理能力的同时提高了 gpu 的灵活性。

4.GPU SM

GPU 的线程调度主要由 SM (流式多处理器，核心计算单元，内部结构很多，接下来列举说明）中的 warp scheduler 负责。

Instruction cache(指令缓存)：存放指令的缓存空间，减少 SM 从全局内存获取指令的延迟——不同的 warp 可以复用已经缓存好的指令，利用时间局部性（短时间内可能重复执行同一条指令）和空间局部性（程序的指令之间是连续的）。

Warp Scheduler(Warp调度器)：决定每个时钟周期要执行的 warp ，当一个warp 由于内存访问或数据依赖而无法执行时，Warp Scheduler 会调度其他 warp 继续执行，确保 SM 内的计算单元尽可能不空闲，最大化资源利用率，同一个 warp 的线程同步执行同一条指令。

Dispatch Unit(指令分发单元)：负责将 Warp Scheduler 选择的指令分发到合适的计算单元（如CUDA Cores、Tensor Cores）上的模块，Dispatch Unit 根据 Warp Scheduler 的决定，从 Instruction Cache 中获取已经调度好的指令，将指令发送到相应的硬件执行单元（如CUDA Core、ALU、Load/Store单元等）。它协调执行不同类型的指令，确保每个硬件单元处理适合其功能的指令。

SP(流处理器)：通用执行单元，也叫 CUDA core，对应逻辑结构的一个 thread，它负责执行具体的加减乘除、加载存储等运算指令。

Tensor Core：专用执行单元，专用于矩阵乘法

整个 SM 内部的协作过程：Instruction Cache 提供已经缓存的指令，减少从全局内存加载指令的延迟，确保 Warp Scheduler能够快速获取所需指令。Warp Scheduler 从 Instruction Cache 中提取指令，根据当前的warp状态和依赖关系，选择合适的warp进行执行。Dispatch Unit 从Warp Scheduler那里获取指令，并将其分发到 执行单元。Instruction Cache的作用是让Dispatch Unit能够快速获得已经缓存的指令，减少指令调度延迟。

5.GPU 内存层次及每种存储的特点

首先有个计算机基本概念，距离执行计算单元越近的内存越宝贵，读写速度越快，尺寸也越小。

  • 全局内存(Global Memory)：GPU 上最大的内存，即常说的 HBM 内存，可以被所有线程访问，从 host 端搬运的数据存放在此处。
  • L2 Cache：SM 访问交互的缓存存放区域，即会放 global/local memory 的数据，当 thread 需要数据时，会先访问 L2 Cache，若没有才访问 global memory， 这时候的读取不会只读单独的数据，而是把周围一部分数据一起带到 L2 Cache上，方便之后或者临近的 thread 能够在L2 Cache上找到它（局部性原理）。
  • 共享内存(Shared Memory)/L1 Cache：位于 sm 内部，同一个 block 的线程可访问同一块共享内存，在物理结构上和L1 cache 位于同一块物理内存，但 shmem 可由程序员手动管理， L1 cache 由硬件自动管理。
  • 本地内存(Local Memory)：每个线程都可以使用自己的本地内存，可以在其中存储临时变量。它具有最小的范围并且专用于每个单独的线程。
  • Register Cache：寄存器是每个线程私有的，访问速度最快，每个SM中，寄存器缓存（Register Cache）用于提供线程的寄存器访问。每个SM包含一定数量的寄存器（physical registers），所有线程都可以从这些寄存器中分配资源。

6.CUDA stream

stream 相等于是 GPU 上的任务队列，即命令流水线，它允许多个 CUDA 操作在不同的 stream 中并行执行，从而提高 GPU 的利用率和性能。一个 stream 上的任务是严格顺序执行的，上一个命令执行结束才会执行下一个命令，不同 stream 的命令之间无执行顺序，即可以并发执行不同命令。

7.CPU、GPU、NPU的区别

CPU 有更多的控制单元，更大的缓存，更少的 ALU(算术逻辑单元)，延迟较低；相较之下，GPU 使用大量的 ALU 来最大化计算能力和吞吐量，少量的缓存和控制单元，延迟较高。

所以对于他们的设计目标来说，CPU 被设计为顺序执行指令，为了提高顺序执行性能，多年来 CPU 设计中引入了许多功能，包括指令流水线、乱序执行、推测执行和多级缓存等。CPU 的重点是减少指令执行延迟，以便 CPU 能够尽快执行指令序列。

GPU 专为大规模并行性和高吞吐量而设计，但代价是较高的指令延迟。

NPU，即神经网络加速器，在电路层模拟神经元，相比于 CPU 和 GPU，NPU 通过突出权重实现存储和计算一体化，从而提高运行效率，一条指令完成一组神经元的处理，提高运行效率

8.DMA RDMA

DMA(直接内存访问)：传统的内存访问需要通过 CPU 进行数据 copy 来移动数据，通过 CPU 将内存中的 buffer1 移动到 buffer2，而 DMA 模式在 DMA Engine 之间通过硬件直接将 buffer1 移动到 buffer2，不需要 cpu 的参与，减少了 copy 开销。

RDMA：即远程直接内存访问，指多个计算机之间访问使用 DMA，一个主机可以直接访问另一个主机的内存。

使用 RDMA 的优势如下：

  • 零拷贝(Zero-copy) - 应用程序能够直接执行数据传输，在不涉及到网络软件栈的情况下。数据能够被直接发送到缓冲区或者能够直接从缓冲区里接收，而不需要被复制到网络层。
  • 内核旁路(Kernel bypass) - 应用程序可以直接在用户态执行数据传输，不需要在内核态与用户态之间做上下文切换。
  • 不需要 CPU 干预(No CPU involvement) - 应用程序可以访问远程主机内存而不消耗远程主机中的任何 CPU。远程主机内存能够被读取而不需要远程主机上的进程（或 CPU)参与。远程主机的 CPU 的缓存(cache)不会被访问的内存内容所填充。
  • 消息基于事务(Message based transactions) - 数据被处理为离散消息而不是流，消除了应用程序将流切割为不同消息/事务的需求。
  • 支持分散/聚合条目(Scatter/gather entries support) - RDMA 原生态支持分散/聚合。也就是说，读取多个内存缓冲区然后作为一个流发出去或者接收一个流然后写入到多个内存缓冲区里去。
